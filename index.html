Identity Threat Hunter

Cloud‑native identity security analytics on Google Cloud Run and Elastic Security.

Open Analyst UI View Source

Overview

ITH ingests authentication activity, enriches events with context and behavior, and raises real‑time alerts for identity‑based threats. The platform is fully serverless on Google Cloud Run and uses Elastic Cloud for analytics and alerting.

  • Seven baseline identity detections
  • Honey Identity canaries
  • Digital Twin enrichment
  • Quantum Guardian (QES) add‑on

Architecture (at a glance)

[User/Judge] → ith-ui (Cloud Run)
                  │
                  ├→ ith-event-gen → ith-ingestor → Elastic (ith-events)
                  │                                ├→ Digital Twin
                  │                                ├→ Honey Identity
                  │                                ├→ Quantum Guardian
                  │                                └→ Kibana Alerts
                  └→ ith-alert (Webhook)

Google Cloud

Cloud Run hosts stateless microservices (UI, ingestor, event‑gen, add‑ons). It scales to zero, supports public endpoints, and integrates with Cloud Build for CI/CD.

  • Serverless containers
  • Secure, public endpoints for demos
  • Cloud Logging for ops visibility

Elastic Security

Elastic Cloud stores events, runs rule schedules, and visualizes alerts. KQL‑based detections provide transparent, reproducible logic.

  • Fast indexing and search
  • Kibana Rules & Alerts
  • Dashboards and timelines
